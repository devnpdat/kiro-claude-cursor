Hermes Agent & 9Router Proxy

Hiểu đúng 2 công cụ trong AI toolkit cá nhân: Hermes làm gì, 9Router giải quyết bài toán gì, và cách setup combo fallback model thực tế.

Hermes Agent Telegram Bot Kimi K2 9Router Proxy Model Combo Fallback Chain ngrok Tunnel Tailscale

❓ Vì sao cần thêm 2 tool này?

  • ❌ Claude Code / Cursor / Kiro — mỗi tool dùng 1 model, không fallback khi rate limit.
  • ❌ Không biết mình đang tốn bao nhiêu token / dollar mỗi ngày.
  • ❌ Không có cách chat với AI từ điện thoại khi không ngồi máy tính.
  • ❌ Khi 1 provider down (Claude, Kiro) — toàn bộ workflow dừng lại.

Concept 1

🦾 Hermes Agent

AI agent chạy local, có TUI, Web dashboard và Telegram bot gateway

WHATLà gì?

Hermes là một AI agent framework chạy trực tiếp trên máy tính của bạn. Không phải SaaS, không phải cloud — là local process với 3 interface: CLI, TUI (terminal UI đẹp), và Web dashboard tại localhost:9119.

WHYTại sao dùng?

Claude Code, Kiro, Cursor đều chỉ chạy trong IDE. Hermes cho phép chat với AI và dùng MCP tools từ bất kỳ đâu — terminal, browser, hoặc Telegram trên điện thoại. Đặc biệt hữu ích khi cần query Jira/Confluence nhanh mà không cần mở IDE.

WHOAi dùng?

Developer cần truy cập AI agent và MCP tools (Jira, Confluence, GitLab) từ nhiều interface khác nhau. Cũng phù hợp khi share tool với teammates qua Telegram bot.

WHENKhi nào dùng?

• Quick query từ điện thoại khi đang họp
• Chạy task AI khi không ngồi trước máy
• Monitor kết quả agent từ Telegram
• Dùng thay Claude.ai khi cần context dự án (Hermes có thể kết nối MCP tools)

WHEREỞ đâu?

• Source: ~/.hermes/hermes-agent/
• Config: ~/.hermes/config.yaml
• Web UI: localhost:9119
• Telegram: bot gateway (tuỳ chọn)
• Gateway: launchd plist (auto-start khi boot)

Concept 2

🔀 9Router Proxy

Local LLM proxy với model combo, fallback chain và usage dashboard

WHATLà gì?

9Router là một local proxy server (chạy tại localhost:20128) expose OpenAI-compatible API. Bạn point các AI tool vào đây thay vì gọi thẳng đến provider. 9Router sẽ route request theo config: single model, combo fallback, hoặc round-robin.

WHYTại sao dùng?

3 lý do chính:
1. Fallback: Khi model A fail/rate limit → tự động thử model B, C...
2. Monitor: Dashboard xem token usage, cost ước tính theo ngày/tuần
3. Unified endpoint: Một URL duy nhất cho tất cả tool, dễ swap model mà không cần cấu hình lại từng tool

WHOAi dùng?

Developer dùng nhiều AI tool (Claude Code, Cursor, Kiro, Hermes) và muốn centralized monitoring + automatic failover. Đặc biệt hữu ích khi dùng nhiều provider khác nhau (Claude Code OAuth, Kiro OAuth, OpenRouter API key).

WHENKhi nào dùng?

• Muốn biết mình đang tốn bao nhiêu token mỗi ngày
• Cần fallback tự động khi một provider down
• Muốn thử nghiệm nhiều model mà không cấu hình lại IDE
• Team muốn dùng chung một endpoint (qua Tunnel)

WHEREỞ đâu?

• Local: http://localhost:20128/v1
• Dashboard: localhost:20128/dashboard
• DB: ~/.9router/db/data.sqlite
• Start: 9router trong terminal
• Providers: Claude Code (cc), Cursor (cu), Kiro (kr), OpenRouter

HOWCách setup

1. Start 9Router

9router
# Chọn: Hide to Tray (Background)
# Server: http://localhost:20128

2. Verify API

# gồm cc/, cu/, kr/, openrouter/

3. Check Usage

# Dashboard → Usage tab
# Total requests, input/output tokens
# Est. cost theo ngày/tuần/tháng

Concept 4

🌐 Tunnel: Expose 9Router ra Internet

Vấn đề: Cursor xử lý request qua server của họ — không reach được localhost

🔍 Root Cause

  • 🖥️ Cursor IDE gửi API request thông qua server của Cursor, không phải local browser
  • 🚫 Server Cursor chặn private network (localhost, 192.168.x, 10.x) — error: "Access to private networks is forbidden"
  • ✅ Giải pháp: cần public HTTPS URL tunnel về localhost:20128

ngrok ✓ Dùng được

Free tier, cần tạo account
  • Cài: brew install ngrok
  • Auth: ngrok config add-authtoken <token>
  • Run: ngrok http 20128
  • URL thay đổi mỗi lần restart (free plan)
  • Hoạt động ngay, không cần config phức tạp

Tailscale Funnel ⚠ Khó

Cần account Tailscale, SSL issues
  • Cài app: tailscale.com + brew install tailscale
  • Enable Funnel: tailscale funnel 20128
  • Lỗi: version mismatch brew vs app daemon
  • Lỗi: DNS block Let's Encrypt (nếu dùng VPN công ty)
  • Dùng app CLI: /Applications/Tailscale.app/Contents/MacOS/Tailscale funnel reset

Cloudflare Tunnel ✗ Bị block

Built-in 9Router, không hoạt động
  • 9Router có tích hợp sẵn Cloudflare tunnel
  • Error: "fetch failed" khi register URL
  • Nguyên nhân: outbound network bị block
  • cloudflared binary không có trong PATH

📚 Lessons Learned from Real Setup

✅ Đúng

9Router cần phải start trước

Cursor báo lỗi "Model not found" không phải do model name sai — mà do 9Router chưa chạy. Luôn verify: curl localhost:20128/v1/models

Tailscale App CLI khác brew CLI

Brew install tailscale và Tailscale macOS app có 2 daemon riêng gây version mismatch. Luôn dùng /Applications/Tailscale.app/Contents/MacOS/Tailscale thay vì tailscale command.

9Router API Key bắt buộc khi enable Tunnel

Tunnel (dù Cloudflare hay Tailscale) yêu cầu "Require API key" bật trước mới có thể Enable. Tạo key trước → bật toggle → Enable tunnel.

❌ Anti-patterns

Cursor dùng localhost trực tiếp

Không bao giờ dùng localhost:20128 trong Cursor base URL — Cursor server-side không reach được private network. Phải dùng tunnel URL.

Đóng terminal chạy tunnel

ngrok tunnel tắt ngay khi đóng terminal. Cursor sẽ mất kết nối. Cần giữ terminal mở hoặc chạy ngrok dưới dạng background service.

Quên lưu 9Router API Key

Key chỉ hiển thị 1 lần khi tạo. Lưu ngay vào Keychain hoặc password manager. Nếu mất → phải tạo key mới và update lại Cursor settings.

Concept 5

⚙️ Config từng tool → 9Router

Chi tiết setup Claude Code và Hermes — đã verified hoạt động

🤖

Claude Code

Dùng ANTHROPIC_BASE_URL để redirect toàn bộ API call qua 9Router. Claude Code tôn trọng env var này.

# ~/.zshrc
export ANTHROPIC_BASE_URL="http://localhost:20128/v1"
export ANTHROPIC_API_KEY="<9router-key>"

# ~/.claude/settings.json
{
  "model": "Clade_Cursor_Kiro_Hermes"
}

⚠️ Lưu ý

9Router hỗ trợ cả Anthropic format (/v1/messages) với Authorization: Bearer <key> header. Verify: curl localhost:20128/v1/messages -H "Authorization: Bearer <key>"

🦾

Hermes Agent

Hermes dùng ~/.hermes/config.yaml. Phải chọn provider openai-api (không phải openrouter) vì OpenRouter hardcode endpoint của họ.

# ~/.hermes/config.yaml
model:
  provider: openai-api
  base_url: http://localhost:20128/v1
  default: Clade_Cursor_Kiro_Hermes
  api_mode: chat_completions

auxiliary:
  title_generation:
    provider: openai-api
    base_url: http://localhost:20128/v1
    api_key: <9router-key>
    model: Clade_Cursor_Kiro_Hermes

# ~/.zshrc
export OPENAI_API_KEY="<9router-key>"

💡 Dùng hermes model để config

Chạy hermes model → chọn OpenAI API → nhập base URL http://localhost:20128/v1 → nhập model Clade_Cursor_Kiro_Hermes. Nhanh hơn edit YAML thủ công.

ToolCơ chếEnv varProvider trong config
Cursor Override OpenAI Base URL + ngrok tunnel N/A (UI settings) OpenAI-compatible
Claude Code ANTHROPIC_BASE_URL ANTHROPIC_API_KEY Anthropic format
Hermes config.yaml base_url OPENAI_API_KEY openai-api
Kiro ❌ Không support custom endpoint — hardcoded AWS infrastructure
